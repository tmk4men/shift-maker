# シフト自動作成アプリ（ルールブック方式・ローカル完結）

**▶ そのまま使う： https://tmk4men.github.io/shift-maker/**


労働基準法と現場の運用ルールを **細かいルールブック** として定義し、
それに従って **決定的なアルゴリズム**（AI・機械学習を一切使わない）でシフトを自動作成するWebアプリです。

- **完全ローカル**：データは使っている端末の中だけに保存。サーバーへの送信は一切ありません
- **AI不使用**：同じ入力なら必ず同じシフトが出ます。なぜその人が選ばれたかも説明できます
- **法令はハード制約**：18歳未満の深夜勤務、週40時間、休憩、週1日の休日などは絶対に破りません
- **設定は最小限**：店長が触るのは対象月・勤務区分・必要人数・スタッフだけ
- **ビルド不要**：`index.html` を開くだけ。GitHub Pages にそのまま置けます

## 使い方

### いちばん簡単な流れ

画面の上に手順バーが出ます。未完了のステップに「次にやること」が表示されるので、それに従うだけです。

1. 公開URLを開く（または `index.html` を直接開く）
2. **① 基本設定** … 対象月・勤務区分（早番/遅番/夜勤）・曜日別の必要人数を決める
3. **② 従業員** … スタッフを登録（責任者・有資格者・新人・18歳未満・扶養内 など）
4. **③ 希望を入れる** … 各スタッフの「行ける日と時間」を入れる（下記の方法で手入力を省けます）
5. **［シフトを作成する］** を押す
6. **④ シフト表** … セルをクリックすれば手直しもでき、その場でルール違反を判定
7. **⑤ 集計** … 出勤日数・労働時間・夜勤回数・深夜/時間外・人件費・年収の壁を確認

翌月は［シフト表］タブの **［翌月 ▶］** から。前月のシフトは残るので、
**月をまたぐ連勤・夜勤明け・インターバル**、および **夜勤/土日の公平性の繰り越し** が自動で効きます。

初期状態は空です。動きを見たいときは［① 基本設定］の **［サンプルの店で動きを見る］** で、
10名・1か月分の希望が入ったサンプルを読み込めます（自分のデータは上書きされます）。

### 店長の手入力をなくす（スタッフ入力ページ）

`index.html?input=1` は、**店のデータが何もなくても単体で動く入力専用ページ**です。
スタッフはここに自分の名前と「行ける日・時間」を入れて、ファイルに保存するかコードをコピーします。

```
スタッフ                         責任者
──────────────────────────────────────────
?input=1 を開く
名前と行ける日時を入力
［ファイルに保存］          →   ［希望ファイルを読み込む］
  または［コードでコピー］   →   ［コードを貼り付けて取り込む］
                                  ↓
                                氏名で自動照合して取り込み
                                （複数人分を一度に選べます）
```

URLは［③ 希望を入れる］の **［スタッフ用の入力ページ］** から取得できます。
すでに登録済みのスタッフに直接入力してもらう場合は、**［スタッフ用リンク］**（`?staff=e1`）を使うと、
その人の入力画面と確定後の自分のシフトだけが表示されます。

### 設定にしていない（固定の）動き

店長の手間を減らすため、次の3つは設定項目にせず固定にしています。

| 動き | 理由 |
|---|---|
| **希望が未入力の日は出勤させない** | 「入れていいか分からない人」を勝手に入れないため |
| **必要人数を超える人は入れない** | 人件費が無駄に増えるのを防ぐため |
| **祝日は自動判定** | 振替休日・国民の休日・春分秋分まで計算します |

設定できるのは **対象月・週の始まり（日曜/月曜）・人件費予算・お店の休み** と、勤務区分・必要人数・スタッフだけです。

### お店の休み

［① 基本設定］で **定休日（毎週◯曜）** と **臨時休業日（年末年始・棚卸しなど）** を設定できます。
休みの日は必要人数の設定より優先され、誰も出勤しません。

### 契約で保障した下限

従業員ごとに **最低出勤日数** と **月間の最低時間** を設定できます。
シフトがこれに届かない場合は「◯時間不足」と報告します（休業手当の検討が必要になるため）。
上限側は **最大出勤日数・月間の上限時間・週の上限時間・月間夜勤上限** を設定できます。

### データの持ち運び

ヘッダーの **［書き出し］／［読み込み］** でJSONファイルとして保存・復元できます。
読み込みに失敗しても、今のデータは壊れません。

## ルールブック

考慮しているルールはすべて `rules/` に文書として置いてあります。コードはこの文書と1対1で対応しています。

| ファイル | 内容 |
|---|---|
| [rules/README.md](rules/README.md) | ルール集の索引・ハード/ソフト制約の考え方 |
| [rules/01-legal.md](rules/01-legal.md) | 労働基準法など法令上の制約 |
| [rules/02-operational.md](rules/02-operational.md) | 現場の運用ルール |
| [rules/03-industry.md](rules/03-industry.md) | 業種別（医療・介護・保育・小売・運輸・警備・製造・コールセンター） |
| [rules/04-constraint-model.md](rules/04-constraint-model.md) | 実装用のモデル化（アルゴリズム・緩和順序） |
| [rules/05-real-world-knowledge.md](rules/05-real-world-knowledge.md) | 現場のホンネ（優遇度・新人ペア・交代時間・年収の壁） |

## 対応しているルール（抜粋）

**法令（変更不可・絶対に守る）**

- 1日8時間・週40時間、時間外は月45時間まで（36協定の原則）
- 6時間超45分／8時間超60分の休憩（勤務区分マスタを検証）
- 週1日以上の休日（＝連続勤務は6日まで）
- 18歳未満の深夜勤務禁止・時間外禁止
- 有給/確定休の尊重

**運用（ON/OFF・重み調整が可能）**

- 必要人数の充足、責任者・有資格者の配置
- 新人は教育担当と必ず同じ勤務（時間が重なっていることまで確認）
- 勤務間インターバル（既定11時間）、夜勤明けの翌日は休み
- 最低/最大出勤日数、連勤上限、月間夜勤上限
- 希望休・出勤希望、入力された勤務可能時間（未入力の日は出勤させない＝固定）
- 年収の壁（扶養内）— 上限超過の禁止＋月ごとのペース配分
- 個人ごとの優遇度（多めに入れたい／控えめにしたい。ただし最低日数は必ず確保）
- 相性NG／相性◎、責任者・有資格者の温存、月内のペース配分
- 夜勤・土日祝・労働時間の公平配分、人件費予算

## アルゴリズム

```
Step1  責任者・有資格者がまだいない枠を最優先で埋める（人が尽きて不在の日を作らない）
       次に全枠へ1人目を配り、同じ条件なら候補が少ない苦しい枠から
Step2  ハード制約を1つでも破る人は候補から除外
Step3  ソフト制約のスコアが最良の人を選ぶ（同点はID順で決定的に）
Step4  埋まらない枠は入れ替え（swap）で修復 → 最低日数の未達を補充
Step5  全体を再検証し、違反と不足を一覧化
```

乱数を使わないため、同じ入力からは常に同じシフトが出ます。

## テスト

```bash
npm test          # または node tests/test.js && node tests/ui-smoke.js
```

- `tests/test.js` … 実際の利用場面を想定した16シナリオ（飲食店・小規模カフェ・24時間営業・
  スタッフ提出ベース・人手不足・扶養内・週20時間上限・新人教育・18歳未満・月跨ぎ・大規模30人・
  お盆の希望集中・手動編集で入る違反・役割の兼任者不在・削除の波及・壊れたデータ）。
  生成結果を **ソルバーとは独立に監査** します
- `tests/ui-smoke.js` … 擬似DOMで画面を実際に描画し、全タブ・主要操作・スタッフ専用モード・
  提出コードの往復・localStorage破損からの復帰を確認します

## 設計上の割り切り

- **サーバーがない**ため、スタッフの希望はファイルかコードの受け渡しで集めます（`?input=1`）
- 人件費は概算です（深夜25%・8時間超25%の割増のみ考慮。社会保険料・交通費は含みません）
- 「年収の壁」の金額はハードコードせず、従業員ごとに設定します（制度改正への追従のため）
- 生成は1回あたり数百ミリ秒〜2秒（30人×31日で約1.9秒）。端末側で計算します
- 貪欲法＋局所修復のため、**最適であることは保証しません**（人手不足の報告が「本当に不可能」とは限りません）

## 注意

法令の具体的な数値（時間外の上限、社会保険の適用要件、いわゆる「年収の壁」、最低賃金）は
**改正が頻繁**です。本アプリは設定値として持っていますが、実運用の前に必ず最新の条文・通達を確認してください。
本アプリは労務管理の補助ツールであり、法令適合を保証するものではありません。
