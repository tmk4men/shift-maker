# シフト作成ルール集

シフト作成アプリが考慮すべき制約・ルールの一覧。
実装時はこのルール集を「制約カタログ」として参照し、各制約に ON/OFF・重み・パラメータを持たせる。

## ファイル構成

| ファイル | 内容 |
|---|---|
| [01-legal.md](01-legal.md) | 労働基準法など法令上の制約（原則ハード制約） |
| [02-operational.md](02-operational.md) | 現場の運用ルール（ハード／ソフト混在） |
| [03-industry.md](03-industry.md) | 業種別の追加規制・慣行 |
| [04-constraint-model.md](04-constraint-model.md) | アプリ実装用のモデル化（データ項目・優先度・緩和順序・目的関数） |
| [05-real-world-knowledge.md](05-real-world-knowledge.md) | 現場のホンネ要件（優遇度・新人ペア・交代時間・年収の壁など） |

## 制約IDの体系

| 接頭辞 | 意味 |
|---|---|
| `LAW-xxx` | 法令由来 |
| `OPS-xxx` | 運用ルール |
| `IND-xxx` | 業種固有 |

## ハード制約 / ソフト制約

- **ハード制約（違反不可）**: 法令、契約上の所定、必要人数の最低ライン。1件でも違反したらそのシフト案は不採用。
- **ソフト制約（できるだけ）**: 公平性、希望休、偏りの是正など。違反にペナルティ点を付け、合計点の最小化を目指す。

同じ項目でも運用によってハード/ソフトが変わる（例：希望休を「絶対」とする職場と「努力目標」とする職場）。
**制約ごとに `hard | soft` を設定で切り替えられる設計にすること。**

## 重要な注意

法令の具体的な数値（上限時間・割増率・社会保険の適用要件・年収の壁・最低賃金）は
**改正が頻繁**。アプリ側ではハードコードせず「法令パラメータ設定ファイル」として外出しし、
改正時に設定値のみ更新できるようにする。実運用前には必ず最新の条文・通達を確認すること。
