# 02. 運用ルール

> ハード/ソフトは職場によって変わる。各制約に `hard | soft` と `weight` を設定できるようにする。
> `✓` = 当初リストにあった項目 / `＋` = 追加項目

## A. 人員充足（需要側）

| ID | ルール | 既定 | 備考 |
|---|---|---|---|
| OPS-001 ✓ | 各時間帯の必要人数を満たす | hard | 時間帯×ポジション別に下限／上限を持つ |
| OPS-002 ＋ | 時間帯別の上限人数（過剰配置の防止） | soft | 人件費に直結 |
| OPS-003 ✓ | 責任者（店長・主任・リーダー）を必ず配置 | hard | 開店〜閉店の全時間帯で最低1名 |
| OPS-004 ✓ | 有資格者を必ず配置 | hard | 例：食品衛生責任者、防火管理者、看護師、介護福祉士、登録販売者 |
| OPS-005 ＋ | 資格の有効期限・更新日を考慮 | hard | 期限切れの資格で配置しない |
| OPS-006 ✓ | 新人だけの勤務にしない | hard | 「独り立ち前」フラグで判定 |
| OPS-007 ✓ | 経験者と新人をバランスよく配置 | soft | 各シフトのスキル合計点に下限を設ける |
| OPS-008 ＋ | ポジション別スキルマトリクス | hard | 人×ポジションで可否（レジ/キッチン/フロア/夜勤 等）。「できない仕事」に入れない |
| OPS-009 ＋ | 開店準備・閉店作業の時間を確保 | hard | 営業時間＝勤務時間ではない |
| OPS-010 ＋ | 引き継ぎ時間の重複配置 | soft | 交代時に15〜30分オーバーラップ |
| OPS-011 ✓ | 繁忙日・閑散日に応じた人員数調整 | soft | 曜日・日付・季節・天候・過去実績 |
| OPS-012 ✓ | イベント・キャンペーン時は増員 | hard | カレンダーイベントテーブルを持つ |
| OPS-013 ＋ | 需要予測との連動 | soft | 客数・売上予測 → 必要人時に変換 |
| OPS-014 ＋ | 休憩中も必要人数を維持 | hard | 休憩時間帯をずらして配置（LAW-023と対） |
| OPS-015 ＋ | ピークタイムの重複配置 | soft | ランチ/ディナー等の山に厚く |
| OPS-016 ✓ | 急な欠勤時の代替要員を確保 | soft | ヘルプ可能者リスト、当日呼び出し可フラグ |
| OPS-017 ＋ | オンコール／待機要員の設定 | soft | |
| OPS-018 ＋ | 研修・OJT期間はトレーナーとペア | hard | 新人の初回N回はトレーナー同時勤務 |
| OPS-019 ＋ | 他店舗応援（ヘルプ）の可否と上限 | soft | 移動時間・交通費も加味 |

| ID | ルール | 既定 | 備考 |
|---|---|---|---|
| OPS-033 ＋ | 本人が入力した勤務可能時間の範囲内でのみ配置する | hard | **入力のない日は出勤させない。本アプリでは設定にせず固定** |
| OPS-030 ✓ | 希望休を反映する | hard/soft | 「絶対休（冠婚葬祭等）」と「できれば休」を区別して重み付け |
| OPS-031 ＋ | 希望出勤（この日は入りたい）を反映 | soft | |
| OPS-032 ＋ | 出勤可能な曜日・時間帯（アベイラビリティ） | hard | 個人ごとのカレンダー |
| OPS-033b ＋ | 固定シフト希望（毎週月曜のみ 等） | hard | |
| OPS-034 ✓ | 最低出勤日数を満たす | hard | 契約上の所定日数 |
| OPS-035 ✓ | 最大出勤日数を超えない | hard | |
| OPS-036 ✓ | 最大連勤日数を超えない | hard | 月をまたぐ連勤も判定すること |

| OPS-039 ＋ | 1回あたりの最低勤務時間 | hard | 例：3時間未満のシフトは作らない |
| OPS-040 ＋ | 1回あたりの最大勤務時間 | hard | |
| OPS-041 ✓ | 契約上の所定労働時間（週20時間 等）を守る | hard | 社保・雇用保険の判定にも直結 |

| OPS-049 ＋ | 試用期間・入社予定日・退職予定日 | hard | 在籍期間外に配置しない |
| OPS-050 ＋ | 休職・長期休暇（産休育休・傷病） | hard | |
| OPS-051 ＋ | 長期休暇期間（年末年始・お盆・GW）の希望調整 | soft | 全員の希望が集中するため別ロジックが要る |
| OPS-052 ＋ | 有給申請とシフトの整合 | hard | 申請済み有給日を勤務にしない／年5日消化の計画（LAW-061） |
| OPS-053 ＋ | 本人の希望勤務時間数（月○時間くらい働きたい） | soft | |

## C. 勤務パターン・体調配慮

| ID | ルール | 既定 | 備考 |
|---|---|---|---|
| OPS-060 ✓ | 夜勤明けの翌日は休み | hard | 「明け」を勤務日扱いするか休日扱いするかを定義 |
| OPS-061 ✓ | 勤務間インターバル確保（例11時間以上） | hard | 日跨ぎ勤務の終業時刻ベースで計算 |
| OPS-062 ✓ | 早番→遅番など無理なシフト順を避ける | hard/soft | 禁止遷移テーブル（前日シフト×当日シフト） |
| OPS-063 ✓ | 連続夜勤回数の制限 | hard | 例：3連続まで |
| OPS-064 ✓ | 月間夜勤回数の上限 | hard | |
| OPS-065 ＋ | 月間夜勤回数の**下限** | soft | 夜勤手当を当てにしている人への配慮 |
| OPS-066 ✓ | 連続勤務後は十分な休暇 | hard | 例：5連勤後は2連休 |
| OPS-067 ＋ | 正循環シフト（日勤→準夜→深夜の順） | soft | 交代制勤務の生体リズム配慮。逆循環を避ける |
| OPS-068 ＋ | 同一勤務帯の固定を避ける／固定する | soft | 職場方針で逆になる |
| OPS-069 ＋ | 週末の連続勤務を避ける | soft | |

## D. 公平性・偏りの是正

| ID | ルール | 既定 | 備考 |
|---|---|---|---|
| OPS-080 ✓ | 夜勤の偏りをなくす | soft | |
| OPS-081 ✓ | 土日祝勤務の偏りをなくす | soft | 祝日も忘れずに |
| OPS-082 ✓ | 早番・遅番の偏りをなくす | soft | |
| OPS-083 ✓ | 残業時間の偏りをなくす | soft | |
| OPS-084 ✓ | 月間勤務時間を均等にする | soft | 契約時間で正規化して比較すること |
| OPS-085 ✓ | 希望休の充足率の公平性 | soft | 「毎回同じ人の希望だけ通らない」を防ぐ |
| OPS-086 ＋ | 繁忙日・不人気日（年末年始等）の担当回数の公平性 | soft | |
| OPS-087 ＋ | 公平性の**履歴繰越** | soft | 前月に不利益を被った人を翌月優遇。累積バランス値を人ごとに保持 |
| OPS-088 ＋ | 公平性指標の可視化 | — | 誰がどれだけ損得したかをダッシュボード表示。納得感がクレーム防止になる |

## F. コスト

| ID | ルール | 既定 | 備考 |
|---|---|---|---|
| OPS-110 ✓ | 人件費予算を超えない | hard | 月次・日次の両方で管理 |
| OPS-111 ＋ | 割増賃金の最小化 | soft | 深夜・残業・休日を高コスト人員に寄せない |
| OPS-112 ＋ | 人時売上高・人時生産性の目標達成 | soft | 必要人時＝予測売上÷目標人時売上高 |
| OPS-113 ＋ | 労働分配率・人件費率の目標 | soft | |
| OPS-114 ＋ | 交通費の考慮 | soft | 日額支給なら出勤日数がコストに直結 |
| OPS-115 ＋ | 時給ランクの考慮 | soft | 同条件なら低単価者を優先、等（※不利益取扱いにならない範囲で） |
| OPS-116 ＋ | 助成金・加算要件の維持 | hard | 介護・保育などは配置基準が加算収入に直結（03参照） |

## G. 運用プロセス（アプリ機能として必要）

| ID | ルール | 備考 |
|---|---|---|
| OPS-130 ＋ | 希望提出の締切日・確定公開日を管理 | LAW-072（通知期限）と対応 |
| OPS-131 ＋ | シフト確定後の変更・交代フロー | 本人間の交代申請 → 承認 → 再検証 |
| OPS-132 ＋ | 確定後の変更は再度すべての制約を検証 | 手動編集で法令違反が入るのを防ぐ |
| OPS-133 ＋ | 欠勤・遅刻発生時の代替候補の自動提示 | 制約を満たす候補を残業余力順に提示 |
| OPS-134 ＋ | 変更履歴・監査ログ | 誰がいつ何を変えたか |
| OPS-135 ＋ | シフトの通知（アプリ/メール/掲示） | |
| OPS-136 ＋ | 制約違反の警告表示と理由説明 | 「なぜこの人が入れないか」を説明できること |
| OPS-137 ＋ | 実現不可能時の緩和提案 | どのソフト制約を何件破れば解けるかを提示（04参照） |
| OPS-138 ＋ | 前月末・翌月頭との連続性 | 月跨ぎの連勤・インターバル・夜勤明けを判定するため前後月のデータを読む |
| OPS-139 ＋ | 実績（打刻）との突合 | 予定シフトと実労働時間の差分管理。上限規制の判定は実績ベース |
| OPS-140 ＋ | 見込み残業時間のアラート | 月中で上限に近づいた人を警告 |
